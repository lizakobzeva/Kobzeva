# Лабораторная работа 4

## Цель работы
Изучить проблему синхронизации потоĸов, научиться выявлять и предотвращать взаимоблоĸировĸи (deadlock).

## Требования
- .NET 8.0 или выше
- Нет внешних зависимостей, кроме System.Collections и xUnit

## 1-ая задача: обедающие философы
Философы думают и едят. Для еды им нужно взять 2 вилки: слева и справа от себя, 
но вилок на всех не хватает, приходится конкурировать за них.

- **DinningProfessorsWithDeadlock**: Решение, попадающее в deadlock.
- **DinningProfessorsWithDouteadlock**: Решение, не попадающее в deadlock 
(всегда слева меньшая по хэшу вилка, а справа - большая, поэтому не возьмут одну и ту же).

## 2-ая задача: о спящем парикмахере
В парикмахерскую приходят клиенты и ждут своей очереди, если нет места в очереди - уходят.
Барбер спит в свободное время и стрижёт клиентов, если они есть в очереди.

В классе SleepingBarber есть 2 соответствующих метода:
Barber (для создания потока барбера) и Customer (для создания потока клиента)

В решении используется ConcurrentQueue для очереди, SemaphoreSlim для захвата барберов клиентов
и Mutex для эксклюзивного изменения данных.

## 3-ая задача: производитель-потребитель
Производители добавляют товары в буфер, потребители забирают из него с помощью BlockingCollection.

В классе ProducerConsumer есть 2 соответствующих метода: 
Producer (для создания потока производителя) и Consumer (для создания потока потребителя)

## Сборка и запуск
- Скомпилируйте проект с помощью dotnet build, а затем запустите исполняемый файл для использования.
- Запустите тесты с помощью dotnet test.
